# 现场演示操作清单（DEMO.md）

> 上台照着这份念 + 操作。目标：5 分钟讲清"痛点 → 断网 → 自修复高光 → 产物 → ROI"。
> 演示统一用界面的**「离线回放」模式**，保证 100% 稳定复现，不赌模型/网络。

## 一、开场（0:00–0:30）痛点

> 念："央国企正在被强制做信创国产化替代，几十万行老代码要从 Oracle 迁到国产数据库。
> 人工迁移按人年算、外包要几百万。但有一道红线——**代码是敏感资产，不许上云**，
> 所以 Copilot、通义灵码云版这些云端 AI 工具，法律上根本进不来。"

## 二、断网（0:30–1:00）—— 全场最强一击

> 动作：**当众拔掉网线 / 关闭 WiFi**（或展示飞行模式图标）
> 念："接下来我全程断网演示。我们的 Agent 和模型都跑在内网，断网照样工作——
> 这正是央国企唯一能用的形态。"

## 三、展示原始代码（1:00–1:30）

> 动作：界面左侧已显示待迁移的 Oracle SQL
> 念："这是一段真实风格的 Oracle 老代码，里面埋了 7 个不兼容点：
> 空值函数、条件函数、外连接、行号、伪表……人工要逐条改、逐条测。"
> 动作：点 **🚀 开始迁移**

## 四、看 Agent 自主干活（1:30–2:30）

> 念（指着右侧时间线）："看，它在自己一步步来——先扫描、识别不兼容点、
> 按风险排序、查迁移知识库、逐项改写。这不是一问一答，是它自己决定下一步做什么。"

## 五、★ 高光：失败 → 自修复（2:30–3:30）★ 全场重点

> 动作：界面顶部高光区会弹出"模型犯错 → 自我修复"的左右对比卡片
> 念（放慢，强调）："注意这里——处理外连接时，**端侧模型第一次改错了，验证没通过**。
> 但它读懂了报错，**自己把 (+) 改成了标准 LEFT JOIN，再验证，通过了**。
> 这就是关键：端侧模型比 GPT-4 弱、会犯错，**让它犯错后还能自己改对，才是我们的工程价值。
> 云端套壳谁都会做，难的是让弱模型在内网把活干对。**"

## 六、人工确认（3:30–4:00）

> 念："遇到高风险项——外连接会影响结果集语义——它没有擅自决定，而是停下来请人确认。
> 企业要的就是这种可控性。"

## 七、产物（4:00–4:30）

> 动作：滚到底部"③ 迁移产物"，展示迁移后 SQL + 迁移报告表
> 念："最后产出迁移后的代码，和一份报告：每一项改了什么、为什么、风险等级。
> 全程，没有一个字节离开内网。"

## 八、收尾砸 ROI（4:30–5:00）

> 念："自动化覆盖 60–80% 的机械迁移，人只 review 高风险项，周期和成本大幅下降。
> 商业模式是私有化部署 License + 迁移项目费。信创是国家级强制工程，
> 而'不许上云'这条红线，把云端巨头全挡在门外——**我们做的，是别人法律上做不了的生意。**"

## 实时模型实测结论（已在 Apple M5 / qwen2.5-coder:7b 上验证）

> 这些是真机跑出来的结论，演示时可据此扬长避短。

**能力边界**：
- ✅ 简单确定的迁移（NVL/SYSDATE/DECODE/(+)外连接/ROWNUM）→ 能自修复改对，会触发"验证失败→修复→通过"完整闭环
- ✅ 复杂高风险项（CONNECT BY 递归 / MERGE 合并 / ROWID）→ **不瞎改，主动升级到人工确认**（这是卖点，不是缺点！）
- ⚠️ 7b 多步自主偶尔不稳定（如把 read 当 list、或提前收尾）→ **上台用「离线回放」最稳**

**实时模式跑通的最佳任务措辞**（措辞越明确，7b 越听话）：
```
基础场景：
用 read_file 工具读取 data/demo_project/legacy_oracle.sql 的内容，再用 grep_dialect 扫描
Oracle 不兼容语法，逐项查规则迁移到达梦并用 run_validation 验证，验证失败时按报错修复指令
重写完整SQL，全部通过后输出迁移报告

进阶场景（展示高风险升级人工确认）：
用 read_file 工具读取 data/demo_project/legacy_oracle_advanced.sql 的内容，再用 grep_dialect
扫描，逐项查规则迁移，高风险项用 request_human_review 请人确认，全部完成后输出迁移报告
```

**演示话术升级**（把"7b会犯错"转成卖点）：
> "注意它对递归查询、MERGE 这种复杂结构，没有瞎改，而是主动停下来请人确认——
> 企业要的不是一个乱改代码的黑盒，是一个**知道自己能力边界、该升级时升级**的可控助手。"
