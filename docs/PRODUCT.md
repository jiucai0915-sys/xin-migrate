# 信迁 Agent（XinMigrate）· 产品介绍

> 纯内网运行的国产化迁移 Agent —— 代码不出域，迁移自动化。

---

## 一、产品背景

**信创国产化替代是一项国家级强制工程。** 央国企、政府、金融等关键行业，正被要求把核心系统从国外技术栈迁移到国产化技术栈：

- **数据库**：Oracle / SQL Server → 达梦、人大金仓、OceanBase
- **基础设施**：x86 → 鲲鹏 / 昇腾 ARM，Windows → 麒麟 / 统信 OS

但迁移面临三重现实困境：

1. **量大、价高、缺人**：一个中型系统几十万到几百万行代码，人工迁移按"人年"计，外包报价动辄数百万，且懂双方技术栈的人才稀缺。
2. **一道不可逾越的红线——代码不许上云**：在这些单位，源代码是核心敏感资产，**法律与合规明令禁止上传至公有云**。
3. **结果**：GitHub Copilot、通义灵码云版等所有依赖云端大模型的 AI 编码工具，**在这个场景里从根本上无法使用**——不是不好用，是合规上根本进不来。

**市场结论**：这是一个由政策强制驱动、需求确定、且把云端 AI 巨头天然挡在门外的市场。谁能提供"纯内网可用"的迁移自动化能力，谁就能吃下这块只有本地化方案才能服务的蛋糕。

---

## 二、产品定位

**一个完全运行在企业内网 / 端侧模型上的智能迁移 Agent。**

你把老代码交给它，它像一位资深迁移工程师那样**自主完成**：扫描不兼容点 → 规划 → 改写 → 验证 → 自我修复 → 高风险项请人确认 → 产出迁移代码与报告。

形态类似 Copilot / Claude Code，但有两个本质区别：**① 全程断网可用；② 专攻"信创迁移"这一垂直场景。**

---

## 三、产品解决了什么（核心功能）

| 功能 | 解决的问题 |
|---|---|
| **自动识别不兼容点** | 自动扫描 Oracle 特有语法（NVL、DECODE、(+)外连接、ROWNUM、CONNECT BY、MERGE、ROWID 等），不再靠人逐行排查 |
| **按规则智能迁移** | 内置迁移知识库，把每个不兼容点改写为国产库等效语法 |
| **验证 + 自修复闭环** | 改写后自动验证；若出错，读懂报错并自我修正，直到通过——无需人盯着改 |
| **数据级语义验证** | 不止"语法对"，还在样本数据上真实执行、比对结果，抓出"语法对但语义错"的隐蔽 bug（如误用 INNER JOIN 丢数据） |
| **高风险项人工确认** | 对递归查询、合并语句、物理行删除等高风险改写，主动停下来请人确认，绝不擅自决定 |
| **产出可追溯报告** | 输出迁移后代码 + 迁移报告（每项改了什么、为什么、风险等级），全程可审计 |
| **全程内网离线** | 模型、代码、数据一个字节都不出企业内网 |

**价值量化**：自动化覆盖 60–80% 的机械迁移工作，人只需 review 高风险项，迁移周期与成本大幅下降。

## 四、产品特性

1. **断网可用**：基于端侧开源模型（Qwen2.5-Coder 等）+ 本地推理，拔掉网线照常工作——这是央国企唯一能用的形态。
2. **弱模型 + 强编排**：端侧模型比云端弱、会犯错。产品的核心工程价值在于用「验证 + 自修复 + 修复指令兜底 + 高风险升级人工」这套编排，**让弱模型在内网把活干对**。
3. **两道质量关**：语法关（残留语法检测）+ 语义关（数据级结果比对），不是改完就完事。
4. **知道能力边界**：复杂高风险结构不瞎改，主动升级人工确认——是个可控的助手，不是乱改代码的黑盒。
5. **垂直知识库可沉淀**：迁移规则库随项目积累越来越厚，形成数据壁垒。
6. **演示/生产双形态**：支持对接真实达梦/金仓验证；离线环境自动降级，保证随时可用。

## 五、为什么选择这款产品

> **一句话：我们做的，是别人法律上做不了的生意。**

| 对手 | 为什么不行 / 我们为什么赢 |
|---|---|
| **云端 AI 编码工具**（Copilot、通义灵码云版等） | 代码不许上云，**合规上根本进不来**。我们纯内网，是唯一可选项 |
| **通用私有化编码助手** | 它们是泛用工具；我们**专攻信创迁移**，沉淀垂直知识库 + 验证闭环，迁移准确率才是关键 |
| **人工 / 外包迁移** | 按人年计、成本数百万、周期数月；我们把机械工作自动化，人只管高风险项 |
| **纯脚本批量替换** | 规则替换处理不了语义（外连接方向、层级查询语义）；我们能理解上下文、自修复、并做数据级语义校验 |

**三个选择我们的理由**：
1. **唯一合规可用**：在"代码不出域"的硬约束下，端侧 Agent 是唯一形态。
2. **真能干对活**：弱模型靠强编排兜底，加上语法+语义双重验证，不是演示玩具。
3. **可控可信**：高风险升级人工 + 全程可审计报告，符合央国企对可控性的刚需。

## 六、商业模式

- **私有化部署 License** + 按系统/按行计费的**迁移项目费**
- 随客户项目沉淀**行业迁移知识库**，形成长期壁垒
- 目标客户：央国企、政府、金融——付费能力强、需求由政策强制驱动
