## 第 2 页 · 痛点（开场即砸）
**标题**：央国企正在被一道红线卡住

- 信创国产化强制替代：Oracle→国产库、x86→国产芯片，**几十万行老代码要改**
- 现状：人工迁移按**人年**计，外包动辄**数百万**，周期数月
- **致命红线**：代码是核心敏感资产，**法律不许上云**
  → GitHub Copilot、通义灵码云版等所有云端 AI 工具，**法律上就进不来**

**一句话**：迁移是央国企现在就在烧钱做的事，而"不许上云"把所有云端 AI 挡在门外。

---

## 第 3 页 · 我们的方案
**标题**：一个跑在企业内网、断网可用的迁移 Agent

- 喂给它老代码 → 它**自主**完成：扫描 → 规划 → 改写 → 验证 → 自修复 → 人工确认 → 产出报告
- 像 Codex / Claude Code，但**断网也能跑**，且专攻"信创迁移"
- 模型、数据、代码**全程不出企业内网**

**视觉**：架构图（内网框 + 端侧模型 + Agent 编排 + 工具 + "全程无外网"标注）

---

## 第 4 页 · 为什么是 Agent，而不是套壳 GPT
**标题**：它会自己决定下一步做什么

- 多步自主：自己决定先扫哪、查哪条规则、改哪项、何时验证
- 工具调用：读文件 / 方言扫描 / 查迁移知识库 / 跑验证 / 请人确认
- **核心：验证→自修复循环** —— 改错了自己跑验证、读报错、改对为止

**一句话**：不是一问一答，是一个会自我纠错的"迁移工程师"。

---

## 第 5 页 · 核心差异化（最重要一页）
**标题**：模型越弱，编排越值钱

- 端侧开源模型（Qwen2.5-Coder）比 GPT-4 弱、**会犯错**
- 云端套壳谁都会做（接个强模型即可）；**难的是让弱模型在内网把活干对**
- 我们的工程价值 = **验证 + 自修复闭环**：
  失败 → 把报错喂回模型 → 自我修正 → 再验证 → 通过
- **这才是护城河，不是套壳**

**视觉**：❌ 错误SQL+报错 → 🔧 自我修正 → ✅ 通过 的三步对比

## 第 7 页 · 技术架构
**标题**：轻、稳、全离线

- 端侧模型：Qwen2.5-Coder，Ollama 本地服务（OpenAI 兼容）
- Agent：自研轻量 ReAct 循环（不依赖重框架）
- 验证：达梦/金仓社区版；离线降级为静态语法校验
- 知识库：Oracle→国产库迁移规则库（可持续沉淀 = 壁垒）

## 第 8 页 · ROI / 商业价值
**标题**：做别人法律上做不了的生意

- 效率：自动化覆盖 60–80% 机械迁移，人只 review 高风险项 → 周期与成本大幅下降
- 商业模式：私有化部署 License + 按系统/按行迁移项目费
- 壁垒：迁移知识库越用越厚 + 垂直 know-how
- 市场：信创是国家级强制工程，央国企/政府/金融全是客户，且"不许上云"天然挡住云端巨头

## 第 10 页 · Q&A 预案（背熟，不放PPT）
| 提问 | 应答 |
|---|---|
| 通义灵码也有私有版，凭什么是你？ | 它们是通用编码助手，我们专攻"信创迁移"垂直，沉淀迁移知识库+验证闭环，迁移准确率才是关键 |
| 本地模型质量不够怎么办？ | 价值正是"自我验证+自修复"让弱模型干对活，现场已演示；模型随国产模型进步无缝升级 |
| 怎么保证迁移正确？ | 验证循环 + 高风险项人工确认，不是黑盒一把梭 |
| 商业上能赚钱吗？ | 私有化 License + 项目费，信创政策刚需，央国企付费能力强 |
| 和直接写脚本批量替换有何区别？ | 规则替换无法处理语义（如外连接方向、层级查询），Agent 能理解上下文并自修复 |

### Q&A 扩展（真机实测后补充的更刁钻问题）
| 提问 | 应答 |
|---|---|
| 你这模型才 7B，能力够吗？万一改错了呢？ | 三层兜底：①验证循环——改完必跑校验，错了自己改；②高风险项强制人工确认，不瞎改；③模型可随国产模型进步无缝换大。我们卖的是这套**编排+兜底**，不是模型本身 |
| 复杂的递归查询、MERGE 它真能改？ | 实测它对这类**主动升级人工确认**，而非乱改。这是设计——企业要的是知道边界、该升级就升级的可控助手，不是黑盒 |
| 验证用的是真数据库吗？ | 支持对接真实达梦/金仓做语法校验；演示环境用静态校验+残留语法检测降级，保证断网可演。生产部署接真库 |
| 迁移准确率多少？ | 机械迁移项（函数/语法）准确率高且可验证；复杂业务逻辑项走人工确认。我们不夸大"全自动"，而是"自动化+人工兜底"的务实组合 |
| 为什么不等更强的国产模型出来再做？ | 编排层与模型解耦，模型越强我们效果越好，但**护城河是迁移知识库+垂直工程**，不依赖某个模型 |
| 怎么防止它把能跑但语义错的代码也"验证通过"？ | 静态校验保语法，语义靠①迁移知识库的规范改写②高风险项人工确认。两道关，不靠模型自觉 |
