# 汇报网页（发布会风）· web/site/

> 答辩开场用的「讲故事舞台」——纯前端、零依赖、断网可跑。
> 与现场 Demo（`web/app.py` 的 streamlit 界面）分工：本页负责讲故事，Demo 负责跑真迁移。

## 怎么打开

- **最简单**：双击 `index.html`，用浏览器（Chrome / Edge）打开即可。
- **全屏演示**：打开后按 `F11` 进入全屏，干净无浏览器边框。
- **可断网**：所有 CSS / JS / 二维码生成器 / Logo 全部内联在单文件里，不联外网。

## 怎么操作（翻页 / 滚动两用）

| 操作 | 效果 |
|---|---|
| `空格` / `↓` / `→` / `PageDown` | 下一屏 |
| `↑` / `←` / `PageUp` | 上一屏 |
| `Home` / `End` | 跳到首屏 / 末屏 |
| 鼠标滚轮 | 整屏吸附滚动（scroll-snap） |
| 右侧圆点 | 点击直接跳到对应屏 |

进入第 6 屏「核心差异化」时，会自动播放**「改错 → 读报错 → 自修复」**的 diff 动画（红框抖动 → 绿框逐行高亮 → ✅ PASS 印章）。

## 10 屏结构

封面 → 痛点 → 致命一击（云端AI进不来）→ 方案 → 为什么是Agent → **★核心差异化(自修复动画)** → 两道质量关 → 技术架构 → ROI/商业 → 路线图+团队署名（含仓库二维码）。

## 内容来源

文案对齐 `docs/PITCH.md`、`docs/PRODUCT.md`、`docs/ROADMAP.md`；视觉与 streamlit 演示界面同一套深色科技配色（青绿 `#00E0A4` + 近黑 `#0B0F14`）。

## 改内容

直接编辑 `index.html` 即可（单文件）。文案在各 `<section class="slide">` 里；配色在顶部 `:root` CSS 变量；二维码地址在脚本里的 `drawQR()` 函数（当前指向 Gitee 仓库）。
